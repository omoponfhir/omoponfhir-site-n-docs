What is OMOPonFHIR2.0?

General Overview:

OMOPonFHIR 2.0 builds on the original version of OMOPonFHIR in hopes of providing a more customizable mapping solution for a broader user base.

Architecture:

The OMOPonFHIR 2.0 package will available in Java and .NET.

Streamlined Configuration:

In OMOPonFHIR 1.0, the issue of maintaining distinct mapping layers between an N times N number of versions between FHIR and the OMOP CDM can create some complexity for developers. With OMOPonFHIR 2.0, mapping is handled through fully configurable JSON files enabling far more reliability and flexibility, as well as simpler maintenance.
